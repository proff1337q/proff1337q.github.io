Лабораторная работа 1

1. Идентификатор прецедента
Автоматизированный процесс верификации

2. Название прецедента
Верификация комплектующей вычислительной системы

3. Контекст
Сборочное производство вычислительных систем

4. Участники и цели
Участник - Категория - Цель
Комплектующая ВС - Основной - Верифицировать на основе исходных требований
Оператор - Внешний - Контролировать процессы на сборочном производстве
Заказчик - Внешний - Установить требования для готовой продукции
Программист - Инструмент - Реализовать автоматизированный процесс верификации
Компьютерная система - Инструмент - Платформа для работы программиста

5. Предусловия
* Проанализировать технические свойства современных комплектующих
* Проанализировать алгоритм сборочного производства
* Освоить навыки кодирования для реализации платформы на языке .Net
* Освоить навыки внедрения платформы в систему 1С производства

6. Пост-условия
* Составленный перечень кодовых слов для определения производителей и типов комплектующих вычислительных систем
* Реализованный автоматизированный процесс верификации на языке .Net
* Внедрение программного продукта в производственный цикл
* Программный продукт работает в автоматическом режиме, что позволяет сократить время на процесс верификации

7. Основной поток
Участник - Действие - Результат
Заказчик - Составляет техническое задание - Готовые требования для разработки автоматизированного процесса
Оператор - Собирает информацию о комплектующих ВС - База данных с наименованиями производителей и техническими характеристиками комплектующих
Программист - Разрабатывает автоматизированную систему на основе полученных данных - Готовый программный продукт для автоматического процесса верификации
Программист - Внедряет ПП в производство - Измененный цикл производство с автоматическим процессом верификации

8. Исключения
* Отсутствие данных о комплектующих конкретного производителя
* Невозможность внедрения в производственный цикл

9. Альтернативы
* Необходимость в составлении более объемной базы данных комплектующих
